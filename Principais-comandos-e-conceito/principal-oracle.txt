Arquitetura: Estrutura  - Resumo 

#Estrutra Logica 
# data blocks, Extents, Segments, Tablespaces

# Data Blocks: Menor unidade logica de armazenamento do banco de dados 

#Extents composta de vários blocos de dados contíguos, tamanho do conjunto de data blocks tambem

#Sagments -conjunto de Extents que contém todos os dados para uma estrutura de armazenamento lógico específica em um espaço de tabela. Obejtos: tabelas,views 

#Tablespaces -Sagments semelhantes são agrupados, facilitando o gerenciamento 






#Fisica 
# Datafiles, Redologs e Controlfiles
 
# Datafiles - Um espaço de tabela em um banco de dados Oracle consiste em um ou mais arquivos de dados físicos. Um arquivo de dados pode ser associado a apenas um espaço de tabela e apenas um banco de dados.
#permitem que eles sejam expandidos sempre que o espaço físico do banco de dados terminar;
#DBF

#Redologs - estrutura mais crucial para operações de recuperação armazenam todas as alterações feitas no banco de dados à medida que ocorrem.

#Controlfiles- necessário para o banco de dados iniciar e operar com sucesso.
#Each control file is associated with only one Oracle database.

$ sqlplus system/senha

*estrutua para conseguir administrar os dados, é o q dá significado para as coisas, sem isso é só dados,
ele guarda quando foi modificado, quem modificou e sabe o que esses dados significam 

#O SO que organiza qual espaço esta livre e não, atraves tambem dos filesystems, qual esta ligado com quem, quem criou 
isso é logico isto é so faz sentido para o SO, se não são só dados 

#https://www.hardware.com.br/livros/hardware/estruturas-logicas.html
muito bom sobre HD, e filesystens

#fragmentação, que reduz consideravelmente a velocidade de acesso, já que dados espalhados significam mais movimentos da cabeça de leitura.

Historia e resumo do livro dominado oracle 9i: Modelagem e desenvolvimento:

e o System R, resolveram lançar a sua versao comercial de um produtor similar.

-1999 - oracle 8i
-2001 - oracle 9i

Funcionalidades avançadas em clusterizacao, confiabilidade e performace 


*Mecanismo de tolerancia a falha com uso de servidor standby

PADRONIZAÇÃO: em nomes de campos e restrições de tabela

nomenclatura de campos da tabela:
qualificador_nomedescritivo
qualificador 	significado
cd		codigo
Nm		nome
nr		nuemro 
vi 		valor
qt		quantidade 


exemplos:
cd_cliente
nr_nota
dt_nascimento


nomenclatura de restrições de uma tabela 
tipo de restricao	paroa de nome
chave primaria 		pk_tabela
chave estrangeira 	fk_tabelamae_tabelafilha
chave unica		un_tabela_campo
chave de checagem	ck_tabela_campo

exemplos:
pk_cliente
fk_municipio_cliente
ck_cliente_ie_fumante 



as infos sobre a restricao que foram criadas para uma tabela são armazenadas em uma tabela de controle, atualizada automaticamente 
tabela USER_CONSTRAINTS

SELECT * FROM USER_CONSTAINTS
WHERE TABLE_NAME - 'NOME_DA_TABLEA';

*O NOME DA TABELA DEVE SER DIGITADA EM MAIUSCULO



tabela USER_CONS_COLUMNS - guarda dados sobre colunas envolvendas em cada restricao 
SELECT * FROM USER_CONS_COLUMNS 
WHERE CONSTRAINT_NAME = 'NOME_DA_CONSTRAINT';
